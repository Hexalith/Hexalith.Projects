## Key Insights

1. The single-source-of-truth pattern scales.
   - One pure policy (Story 3.1), one matrix doc (`docs/context-assembly-decision-matrix.md`), and one completeness test enforce a contract that four HTTP surfaces consume verbatim. The marginal cost of adding the fourth surface (Story 3.5) was a strict-subset port plus three wire-shape invariants.
   - When the policy and the matrix doc are wired together by a Tier-1 test, drift becomes a CI failure instead of a code-review observation.

2. Partial-class handler files are the right granularity for grouping HTTP endpoints.
   - `ProjectsDomainServiceEndpoints.cs` is 1999 LOC pre-Story-3.2 and would be unreadable if Epic 3 had added four 150-LOC handlers inline.
   - Each `Queries/Get*Endpoint.cs` partial owns one route, one method, one signature, one test class. The partial-class technique preserves the single class identity (DI, swagger, route registration) while distributing implementation across cohesive files.
   - This is a generalizable pattern for any module whose "endpoints" file passes ~1500 LOC.

3. Capability gates are now mature enough to handle deferred dependencies cleanly.
   - Story 3.4's file-reference HALT did not block the story. The additive interface method shipped; the handler doesn't invoke it; the tests are explicitly deferred; the dependency is named (Folders-side opaque-id read route); the gap is recorded.
   - This is the third consecutive epic where a capability gate produced a graceful degradation (Story 2.4 `ProjectFolderCreationPending`; Story 2.7 Memories-via-NuGet-not-submodule; Story 3.4 file-recheck deferred).
   - The gate is no longer "what blocks this story"; it is "what scope reduction lets this story still ship the core value."

4. The "port the previous handler with documented small deltas" pattern compresses implementation effort.
   - Story 3.3 port from Story 3.2: 2 changes (OperationKind, wire body wrap).
   - Story 3.4 port from Story 3.2: 3 changes (OperationKind, ACL recheck fan-outs, rechecked-state composition).
   - Story 3.5 port from Story 3.2: subset port (no sibling ACL deps) + 1 stricter contract (defensive Unauthorized/ProjectUnavailable safe-denial 404 collapse).
   - The result: each subsequent handler shipped faster than the first, with reviewer effort focused on the documented deltas instead of re-reading the boilerplate.
   - This pattern requires that the first handler in the family is correctly factored. Story 3.2's choice to extract the partial-class file and define `ProjectContextConversationsPageSize` as a shared constant paid back through the rest of the epic.

5. Fail-closed posture continues to be the default at every layer.
   - Outer collapses (Tenant authority missing, project not visible, cross-tenant) safe-deny to 404 BEFORE per-reference checks run.
   - Per-reference checks include with an explicit reason code or exclude with an explicit reason code — never silently include, never silently fail.
   - Story 3.5 added a defense-in-depth `Unauthorized | ProjectUnavailable → 404` collapse on the conversation-start fast path even though the policy already returns the correct outer outcome — stricter than Stories 3.2 / 3.3 / 3.4 by design.
   - The pattern is now: when the wire body is supposed to be a happy-path artifact only (Story 3.5 vs the Problem-Details-or-body bifurcation of 3.2 / 3.3 / 3.4), additionally fail-close at the boundary.

6. Documentation artifacts that ship with the code stay in sync.
   - `docs/context-assembly-decision-matrix.md` is referenced by Tier-1 tests and updated additively by each story.
   - `docs/checklists/mutation-and-query-negative-tests.md` rows are ticked off in each story's Dev Agent Record.
   - `docs/event-catalog.md` "producer of last resort" section names unproduced shared-vocabulary outcomes with planned producers — and Story 3.4 closed `ReferenceState.Stale` for folder/memory (the planned producer per the tracker).
   - This is the second consecutive epic where docs-in-repo with test enforcement beat docs-out-of-repo without it. The Epic 2 retro Action Item 7 prediction held: promote per-story patterns to repo-level checklists.
